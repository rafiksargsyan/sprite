How it works

Three steps from video URL to ready-to-use sprites.

1

Define a job spec

Configure your output formats (WebP, AVIF, JPG, Blurhash), resolutions, sprite grid size, and intervals — once, reuse forever.

→
2

Submit a job

POST a video URL with your spec ID. Sprite fetches the video, runs FFmpeg, and assembles the sprite sheets asynchronously.

→
3

Download & use

Grab the ZIP with sprite images and WebVTT files. Drop them into your player's thumbnail preview configuration and you're done.

Everything you need

Built for developers who don't want to run video infrastructure.

🎞️

Multiple formats

WebP, AVIF, JPG, and Blurhash — generate all of them in a single job from one video URL.

⚡

Async processing

Jobs are queued and processed asynchronously with automatic retries. No request timeouts, no babysitting.

🔑

API key access

Integrate directly from your backend using API keys. No OAuth flow required for server-to-server calls.

📐

Flexible sprite grids

Configure rows, columns, resolution, and interval per format. Fine-tune the trade-off between quality and file size.

👁️

Built-in preview

Scrub through generated thumbnails directly in the dashboard before downloading or deploying.

📦

WebVTT included

Every job produces a WebVTT file alongside the sprites. Compatible with Video.js, Plyr, Shaka Player, and more.
